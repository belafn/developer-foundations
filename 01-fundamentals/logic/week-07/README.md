# Semana 07 — Robustez, Validação e Refatoração de Código

Nesta semana o foco passa a ser a construção de código mais robusto, confiável e sustentável.

Além de escrever lógica funcional, começamos a tratar cenários de erro, validar dados de entrada e melhorar a organização interna do código através de refatoração e separação de responsabilidades.

Esses conceitos aproximam o desenvolvimento de práticas utilizadas em aplicações reais, onde lidar com dados imprevisíveis e manter código legível ao longo do tempo é essencial.

## Tópicos abordados

### 01. Tratamento de erros
- Conceito de falhas em tempo de execução
- Uso de `try` / `catch`
- Lançamento de erros com `throw`
- Criação de mensagens de erro claras
- Prevenção de comportamentos inesperados

### 02. Padrões funcionais
- Uso de funções puras
- Evitar efeitos colaterais
- Encadeamento de funções
- Composição de lógica
- Reutilização de funções

### 03. Validação de dados
- Verificação de tipos e formatos
- Validação de parâmetros de função
- Prevenção de dados inválidos
- Estruturação de funções de validação
- Garantia de consistência nos dados

### 04. Lógica em camadas
- Separação de responsabilidades
- Organização da lógica em níveis
- Camada de validação
- Camada de processamento
- Camada de apresentação de resultados

### 05. Prática de refatoração
- Melhorar código existente sem alterar comportamento
- Reduzir duplicação
- Melhorar legibilidade
- Simplificar estruturas complexas
- Aplicar princípios de Clean Code

### 06. Mini projeto
Aplicação integrada dos conceitos:
- Validação de dados de entrada
- Tratamento de erros
- Organização da lógica em camadas
- Refatoração para melhorar legibilidade
- Estruturação clara das responsabilidades do código

## Objetivos de aprendizado

- Construir código mais resiliente a erros
- Validar dados antes de processá-los
- Organizar lógica em camadas bem definidas
- Melhorar código existente através de refatoração
- Aplicar padrões funcionais para aumentar previsibilidade
- Escrever código mais próximo de padrões profissionais

## Evolução técnica até aqui

- Week 01–03: Fundamentos da linguagem e controle de fluxo
- Week 04: Estruturação com funções e escopo

# Week 07 — Robustness, Validation and Code Refactoring

This week focuses on building more robust, reliable, and maintainable code.

Beyond writing functional logic, we start handling runtime errors, validating input data, and improving code structure through refactoring and separation of responsibilities.

These concepts move development closer to real-world practices, where dealing with unpredictable data and maintaining readable code over time is critical.

## Topics covered

### 01. Error handling
- Runtime failure concepts
- `try` / `catch`
- Throwing errors with `throw`
- Writing clear error messages
- Preventing unexpected behavior

### 02. Functional patterns
- Pure functions
- Avoiding side effects
- Function chaining
- Logic composition
- Function reuse

### 03. Data validation
- Type and format checking
- Function parameter validation
- Preventing invalid data
- Structuring validation functions
- Ensuring data consistency

### 04. Layered logic
- Separation of concerns
- Organizing logic into layers
- Validation layer
- Processing layer
- Result/output layer

### 05. Refactoring practice
- Improving existing code without changing behavior
- Reducing duplication
- Improving readability
- Simplifying complex structures
- Applying Clean Code principles

### 06. Mini project
Integrated application:
- Input data validation
- Error handling
- Layered logic organization
- Code refactoring
- Clear responsibility structure

## Learning goals

- Write code that is resilient to runtime errors
- Validate data before processing
- Structure logic using layered architecture
- Improve existing code through refactoring
- Apply functional patterns for better predictability
- Write code closer to real-world development practices
